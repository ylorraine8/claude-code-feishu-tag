## 实现思路

整体链路是：

飞书群聊
  ↓
飞书自建应用 / 机器人
  ↓
飞书长连接事件
  ↓
本地监听脚本
  ↓
Claude Code
  ↓
飞书群聊 / 本地文件 / 多维表格

简单理解：

飞书负责接收群聊消息，Claude Code 负责理解和执行任务，MCP 负责把两边接起来。

如果你不知道 MCP 是什么，可以先把它理解成一个「万能转接头」。就像 Type-C 转 Lightning 的转接线，MCP 就是让 Claude Code 能直接调用飞书能力的那根线。

## 第一步：创建飞书自建应用

打开飞书开放平台：

[https://open.feishu.cn/](https://open.feishu.cn/)

然后按下面步骤操作：

1. 进入「开发者后台」
2. 点击「创建企业自建应用」
3. 应用名称随便填写，例如：`Claude Code 助手`
4. 创建完成后，进入应用后台
5. 记下两个信息：

```text
App ID
App Secret
```

后面连接 Claude Code 和飞书 MCP 时会用到。

## 第二步：开通飞书权限

这一步最容易踩坑。

我自己搭的时候，权限来来回回加了五次。每次报错都缺一个权限，加上之后又报下一个。所以建议你一次性把下面这些权限全部开通。

进入应用后台的「权限管理」，搜索并开通：

| 权限 | 用途 |
| --- | --- |
| `im:chat` | 创建群聊 |
| `im:message:send_as_bot` | 让机器人发送消息 |
| `im:message.group_msg` | 读取群聊消息 |
| `contact:user.id:readonly` | 通过手机号查找用户 |

### 重要：权限开完以后必须发布版本

权限不是开通后立刻生效。

你必须进入：

```text
版本管理与发布 → 创建新版本 → 发布
```

如果不发布，权限不会生效。

这是飞书最容易坑人的地方。我自己就踩了三次：明明已经加了权限，测试还是报错，最后发现是没有发布新版本。

## 第三步：配置事件与回调

接下来要让飞书把群里的新消息主动推送给你的机器人。

进入：

```text
事件与回调
```

接收方式选择：

```text
使用长连接接收
```

然后添加事件：

```text
im.message.receive_v1
```

这个事件的作用是：当群里有新消息时，飞书会主动通知你的机器人。

添加完成后，再发布一次版本：

```text
版本管理与发布 → 创建新版本 → 发布
```

到这里，飞书侧的基础配置就完成了。

## 第四步：在 Claude Code 中连接飞书 MCP

打开 Claude Code，添加飞书 MCP，并填入刚才保存的：

```text
App ID
App Secret
```

连接成功后，Claude Code 就可以调用飞书能力，例如：

- 发送群消息
- 创建群聊
- 查询用户
- 读取群消息
- 操作飞书文档
- 操作飞书多维表格

你不需要自己去读飞书 API 文档，Claude 会根据你的需求调用对应能力。

## 第五步：测试创建群聊

你可以直接在 Claude Code 里说：

```text
帮我创建一个测试群，然后把我拉进去，我的手机号是 XXX。
```

Claude 会自动完成：

1. 调用飞书接口创建群聊
2. 通过手机号查找你的用户 ID
3. 把你拉进群
4. 往群里发送一条测试消息

如果你打开飞书，看到一个类似「Claude AI 助手测试群」的群，并且里面有 Claude 发来的消息，说明基础链路已经跑通了。

到这一步，你已经实现了 Claude Tag 最基础的能力：AI 可以进入群聊并发送消息。

## 第六步：实现实时双向对话

目前为止，Claude 已经能往飞书发消息。

但如果你在群里说话，Claude 还不会自动回复。原因是它还没有持续监听群消息。

你可以直接对 Claude Code 说：

```text
帮我写一个飞书机器人，能实时监听群里的消息，收到之后自动回复。
```

Claude Code 会帮你完成：

- 编写监听脚本
- 配置长连接
- 接收飞书群消息
- 调用 Claude 生成回复
- 把回复发回飞书群

你不需要自己手写完整代码，只需要描述你想要的效果。

## 第七步：让它保持在线

如果你想让这个 AI 同事 7 × 24 小时在线，可以让你的 Mac 保持运行。

这样你出门在外，也可以用手机在飞书群里 @ Claude。只要你的电脑还在运行监听脚本，Claude 就能继续回复和执行任务。

## 常见坑位

### 1. 权限开了但还是报错

大概率是没有发布新版本。

解决方法：

```text
版本管理与发布 → 创建新版本 → 发布
```

### 2. 群里发消息后 Claude 没反应

检查这几项：

- 是否添加了 `im.message.receive_v1` 事件
- 是否选择了「使用长连接接收」
- 添加事件后是否重新发布版本
- 本地监听脚本是否正在运行

### 3. 机器人不能发消息

检查是否开通：

```text
im:message:send_as_bot
```

并确认已发布新版本。

### 4. 不能通过手机号拉人

检查是否开通：

```text
contact:user.id:readonly
```

并确认手机号和飞书账号绑定一致。

### 5. 只能发消息，不能读群消息

检查是否开通：

```text
im:message.group_msg
```

以及是否配置了消息接收事件。
